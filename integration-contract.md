# integration-contract.md — ТехПоток

Документ фиксирует реализованные интеграции и шаблон дополнительных аналитических событий для заполнения совместно с заказчиком.

---

## 1. Реализованные интеграции

### 1.1 Форма → Email

| Параметр | Значение |
|---|---|
| Метод | `fetch` POST, Content-Type: application/json |
| Endpoint | `VITE_FORM_ENDPOINT` из `.env` |
| Поля | `name`, `phone`, `company`, `message` (необязательные поля опциональны) |
| Состояния | loading / success / error с `aria-live="polite"` |
| Базовые события | `form_submit_success`, `form_submit_error` |

**Что нужно от заказчика:**
- [ ] Задать `VITE_FORM_ENDPOINT` в `.env` (n8n webhook / Make / EmailJS / собственный backend)
- [ ] Проверить CORS-настройки endpoint'а

### 1.2 Яндекс.Метрика

| Параметр | Значение |
|---|---|
| Подключение | Скрипт добавляется вручную в `<head>` index.html после получения ID |
| ID | `VITE_METRIKA_ID` из `.env` |
| Wrapper | `ym(METRIKA_ID, 'reachGoal', name, params)` в `main.js` |

**Что нужно от заказчика:**
- [ ] Передать номер счётчика Яндекс.Метрики
- [ ] Добавить скрипт счётчика в `index.html` (раздел `<head>`) — код берётся из кабинета Метрики

## 2. Базовые аналитические события (реализованы)

| Событие | Триггер |
|---|---|
| `form_submit_success` | Успешная отправка формы (и контактной, и модальной) |
| `form_submit_error` | Ошибка отправки (сеть / сервер) |
| `phone_click` | Клик по любой ссылке `href="tel:..."` |

---

## 3. Дополнительные события (шаблон — заполнить с заказчиком)

> Заполните совместно с заказчиком. Каждое событие — цель, которую нужно создать в Яндекс.Метрике.

| Событие | Триггер | Приоритет | Статус |
|---|---|---|---|
| `hero_cta_click` | Клик «Получить консультацию» в Hero | Высокий | [ ] |
| `modal_open` | Открытие попапа консультации | Высокий | [ ] |
| `case_slide_next` | Переключение слайда кейсов | Средний | [ ] |
| `faq_open` | Открытие вопроса в FAQ | Средний | [ ] |
| `footer_phone_click` | Клик по телефону в footer | Средний | [ ] |
| `scroll_50` | Прокрутка 50% страницы | Низкий | [ ] |
| `scroll_90` | Прокрутка 90% страницы | Низкий | [ ] |
| `[событие]` | [Описание триггера] | [Приоритет] | [ ] |

**Реализация:** после согласования добавить вызов `trackEvent('имя_события')` в нужное место `main.js`.

---

## 4. Серверное логирование согласия (рекомендация)

Frontend-слой фиксирует согласие на клиенте (checkbox checked = submit разблокирован).
По требованиям 152-ФЗ рекомендуется дополнительно логировать согласие на сервере:
- дата/время согласия
- IP-адрес (анонимизированный)
- версия политики конфиденциальности

**Статус:** реализация на стороне backend/endpoint — ответственность заказчика.
